## Overview

### The challenge

Users should be able to:

- View the optimal layout for the site depending on their device's screen size
- Shorten any valid URL
- See a list of their shortened links, even after refreshing the browser
- Copy the shortened link to their clipboard in a single click
- Receive an error message when the `form` is submitted if:
- The `input` field is empty

### What I learned

I learned how to integrate an API into a web application project. I ran into a major problem because integrating the API described in the instructions required backend development skills, which I don’t have yet. I had to find other solutions, and in the end, I integrated a different API of the same type that can be implemented using JavaScript. This experience taught me how to find possible solutions to ensure the project works properly.

### Continued development

I will now focus on development methods to improve project delivery. I will start adopting the mobile-first approach to save time and energy. I will also begin learning back-end development to round out my skills.
